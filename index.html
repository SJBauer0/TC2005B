  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    El Internet es la interconexion de todas las computadores del mundo en un red compleja. En cambio la World Wide Web o WWW es la coleccion de paginas web que se pueden acceder por todo el mundo

  • ¿Cuáles son las partes de un URL?

    Para empezar un URL significa Uniform Resource Locator es la direccion unica de cada de los recursos de la WWW. Las partes del url son los siguiente:

    • Esquema: es el protocolo de inicio empleado para la transmisión de datos entre ordenadores y la recuperación de la información de internet. (http, https)
    • Host: normalmente, para el protocolo http, el host es: www.
    • Dominio: corresponde con el nombre del dominio.
    • Puerto: define el nombre del puerto según el host. Para el protocolo http.(Normalmente se ignora!)
    • Ruta: define el camino que se debe seguir en el servidor..
    • Nombre de archivo: define detalladamente las características del documento.
  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    El propósito de los metodos son para la comunicación entre el cliente o usario al servicio web. El GET nos permite extraer informacion del servicio web. HEAD se parece mucho al GET solo que no se retorna ningun contendido. POST es para publicar y mandar informacion al servicio web y por ultimo PUT lo cual funciona para modificar la informacion del servicio web.

    Si, porque el error 500 significa que el error es del lado del servidor. Lo cual signifca que solamente el desarrollador puede corregir el sitio y el usuario no tiene ninguna responsabilidad sobre error.

  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    HTML5 mejora varios aspectos que eran difíciles de trabajar de HTML4. HTML5 puede hacer muchas cosas de forma nativa, mientras que HTML4 depende por completo de diferentes sistemas. HTML5 es un conjunto de tecnologia que contiene HTML, CSS y JavaScript

  • ¿Qué componentes de estructura y estilo tiene una tabla?

    Un elemento tabel primero necesita un elemento table para contener todos los diferentes atributos. Las atributos necesarios son los siguientes:

    • caption: Provee un título para la tabla.
    • colgroup: agrupa columnas temáticamente.
    • col: aplica atributos comunes y estilos a una o más columnas. Sólo puede ser declarado como hijo de colgroup.
    • tbody: representa a un cuerpo de la tabla.
    • thead: representa la sección de encabezado de la tabla.
    • tfoot: representa la sección de pie de la tabla.
    • tr: representa una fila. Puede ser declarado como hijo de tbody, thead, tfoot o table.
    • td: representa a una celda regular. Sólo puede ser declarado como hijo de tr.
    • th: representa a una celda de encabezado. Sólo puede ser declarado como hijo de tr.

    El ciclo de vida de un sistema de información es un enfoque por fases del análisis y diseño que sostiene que los sistemas son desarrollados de mejor manera mediante el uso de un ciclo especifico de actividades del analista y del usuario.

    El ciclo de vida del desarrollo del sistema (SDLC, Systems Development Life Cycle) se refiere al proceso de planificación, creación, pruebas y despliegue en un sistema de información.
